¿Qué es Git?

Git es un sistema de control de versiones de código abierto y distribuido. Fue creado por Linus Torvalds

en 2005 y se utiliza ampliamente en la industria para el desarrollo de software. Git permite a los

desarrolladores llevar a cabo un seguimiento de los cambios realizados en el código a lo largo del

tiempo, facilitando la colaboración en equipo y la resolución de conflictos.

Con Git, los desarrolladores pueden guardar diferentes versiones de un proyecto y revertir a versiones

previas si es necesario. También pueden trabajar en paralelo en diferentes ramas o versiones del proyecto

y fusionarlas después en una sola versión. Además, Git es muy escalable y eficiente, lo que lo hace ideal

para proyectos de código grandes y complejos.

Inicializar

El comando git init crea un nuevo repositorio de Git. Puede utilizarse para convertir un proyecto

existente y sin versión en un repositorio de Git, o para inicializar un nuevo repositorio vacío.

Estado

El comando git status muestra el estado del directorio de trabajo y el área del entorno de ensayo.

Permite ver los cambios que se han preparado, los que no y los archivos en los que Git no va a realizar

el seguimiento. El resultado del estado no muestra ninguna información relativa al historial del proyecto.

Confirmar

El comando git commit captura una instantánea de los cambios preparados en ese momento del proyecto.

Las instantáneas confirmadas pueden desear como versiones "seguras" de un proyecto: Git no las

cambiará nunca a no ser que se lo pida expresamente.
